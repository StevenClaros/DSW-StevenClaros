Propósito

El propósito del sitio es dar a conocer a los usuarios tanto las definiciones como las aplicaciones de CSS tomando como base su sintaxis y sus selectores, además de dar a conocer como se lleva a cabo la interacción entre humano y ordenador, de esta manera dar solución a la actividad fase 2 - Diseño sitios web.

Sintaxis y selectores CSS

Inicialmente vamos a definir qué es CSS, este es el formato para páginas creadas en formato HTML basada en los estándares de hojas estilo cascada, su uso le permite ahorrar tiempo, dar consistencia y facilitar la escritura de páginas. Según Sirvent Collado, L., & Luján-Mora, S. (2013) la sintaxis consta de dos partes, el selector y la declaración, por ejemplo: h2 {color:black} En éste ejemplo la etiqueta h2 sería el selector, notifica que el estilo definido se aplicaría a los elementos h2 y la declaración color:black indica que el color del texto de tolos los elementos deben ser de color negro. La declaración se debe encerrar entre llaves y contener las definiciones. Cada una consta de propiedad/valor, donde se separa por dos puntos selector {propiedad:valor} El selector es el nombre de un elemento HTML pero también puede ser de clase, un ID de selector o un selector de contexto, si el valor tiene más de una palabra se pone entre comillas. Para aplicar estilos a varios elementos se pueden agrupar los selectores que comparten definiciones. Para ello se escribe un elemento tras otro separado por coma.

Selectores de Clase

Los selectores de clase le permiten establecer distintos estilos a aplicar al mismo elemento, al igual que cualquier atributo se puede especificar una vez en un elemento. También se pueden definir sin el nombre de un elemento en el principio, en este caso las definiciones se aplican a cualquier elemento cuyo atributo de clase tiene el valor.

Selectores de identificación

Es diferente al selector de clase porque se aplica a un elemento de la página. Las reglas dictan que los valores del atributo no se pueden repetir en la misma página. Se desprende que el número de elementos en un determinado documento es uno o es cero.

Selector de Tipo

Son aquellos que selecciona todos los elementos que coninciden con el nombre del elemento especificado

Selector Universal

Este selector selecciona todos los elementos. Tiene la opción de estar restringido a un nombre específico o a todos los espacios de nombres

* ns | **|*

Selector de atributo

Selecciona todos los elementos basándose en el valor de un atributo específico

Selector por pseudoclase :matches()

Es del tipo funcional y admite como argumento una lista de selección, se indica dentro del paréntesis y separados por comas los casos en que aplicará el bloque de declaraciones

Selector por pseudoclase :not()

Al contrario que el selector matches el selector not apunta a los que no cumplen con los argumentos indicados

Selector por pseudoclase :has()

Es la formulación del selector padre, se usa para apuntar a elementos que contengan lo que se incluyo en el argumento

Selector por pseudoElemento

  1. ::after Permite la incersión de contenido adicional antes del contenido de cualquier elemento
  2. ::attr() Obtiene el valor de un atributo para el primer elemento del conjunto coincidente o establece uno o más atributos para cada elemento coincidente
  3. ::backdrop
  4. ::before Al contrario del atributo after este permite la incersión de contenido adicional después del contenido de cualquier elemento
  5. ::choices Se usa para definir opciones de una campo para obtener la lista desplegable
  6. ::first-letter Es usado para acceder a la primera letra del contenido de un elemento
  7. ::first-line Se usa para acceder a la primera línea del contenido de un elemento
  8. ::footnote-call Es insertado en lugar del elemento footnote cuando es removido del flujo. De forma predeterminada el valor de este pseudo elemento es el valor del contador foortnote con el estilo de un número en superindice
  9. ::footnote-marker Representa el marcador del footnote , el número o simbolo que identifica cada uno. Éste se coloca al principio del contenido del padre y está en la línea predeterminadamente
  10. ::marker Es usada para numerar los párrafos designados como "Notes"
  11. ::nth-fragment() Describe algunas de las fragment boxes generadas por un elemento, el argumento toma la misma sintaxis que el argumento nth-child definida en el SELECT, la diferencia radica en que el número es relativo a los cuadros de fragmentos que se generaron por el elemento
  12. ::read-only Representa un elemento no editable por el usuario
  13. ::repeat-index Representa el elemento actual de una secuencia repetitiva
  14. ::repeat-item Representa un elemento único de una secuencia repetitiva. Generada como padre de los elemento en un único elemento repetido. Se genera como padre de los elementos en un elemento repetido, cada uno está sociado con un grupo de datos de instancia particular y se afecta por propiedades de cada elemento del modelo.
  15. ::value

Interacción Humano - Ordenador

Se puede definir como la disciplina encargada de estudiar como las personas interactúan con las computadoras y hasta que punto se desarrollan para facilitar la interacción con las personas denominada HCI, esta consta de tres partes, los usuarios, los ordenadores y la interacción realizada entre ellos.

Ésta se convirtió en una disciplina con la creación de la computadora personal, con el primer Macintosh y Commodore 64 utilizados en oficina, las personas notaron como esta transición afecta no solo al trabajo sino a sus vidad diarias. El HCI es un dominio que abarca disciplinas como la informática, ingenieria de factores humanos, inteligencia artificial, lingüistica, entre otros.

En la interacción las palabras a tener en cuenta son la usabilidad, seguridad y funcionalidad, para realizar la investigación de este debe tener en cuenta lo siguiente:

  1. Comprender los factores que determinan como las personas hacen uso de la tecnología debe ser el punto de partida para cualquiera de los proyectos de HCI. Es un proceso continuo desarrollado a lo largo del proyecto
  2. La segunda parte es el enfoque centrado en las personas. Se expande para crear soluciones para proyectos como accesibilidad para personas mayores o discapacitados. En ésta fase el diseño de interacción consiste en la creación de sistemas encargados de mejorar la vida de las personas.
  3. Se debe partir de las necesidades de cliente, desarrollando herramientas y técnicas usadas para la construcción de sistemas adecuados para la satisfacción de dichas necesidades.
  4. Se debe crear la interacción del sistema de usuario efectiva, eficiente y segura.
Teniendo en cuenta lo anterior se debe empezar por entender las necesidades de las personas, para construir el sistema de acuerdo a las necesidades del usuario

También hay que tener en cuenta que la usabilidad es uno de los conceptos clave. Es decir, hacer que los sistemas sean fáciles de usar. Mejorando la competitividad, fidelización de clientes y reducción de costes. Durante las pruebas de usabilidad se establecen los aspectos positivos y negativos del prototipo. Estas se basan en analizar los factores psicológicos humanos

El HCI está centrado en la investigación para el desarrollo de una comprensión empírica de usuarios. Los diseñarores de la experiencia de usuario se enfocan en la industria constuyendo productos y servicios. Estas disciplinas son complementarios, ya que los diseñadores usan la investigación sobre las mentalidades de los usuarios.

Usos de HCI

Es usada en muchas areas como el comercio e iindustria, las personas se conectan con máquinas para obtener resultados óptimos. Así mismo, la investigación militar aprovecha esta interacción. En el campo de la medicina los dispositivos ayudan pacientes en tareas cotidianas. El objetivo es armonizar las competencias humanas con las ventajas de las maquinas aprovechando las ventajas de ambos.

Por ejemplo empresas como Google usan proyectos que muestran como se observa la interacción entre el ordentador y las personas. Al igual que proyectos como Siri, el control por voz para realizar búsquedas es una de las tecnologías de futuro. Elementos como Google Glass funcionan con comandos de voz. Funciones complementarias de Google Suggest de búsqueda tabién se identifica como parte de HCI. Otro paso más hacía la vinculación es el desarrollo de funciones de búsqueda como partes de distintas actualizaciones.
